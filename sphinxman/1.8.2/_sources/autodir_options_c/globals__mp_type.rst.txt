:term:`MP_TYPE <MP_TYPE (GLOBALS)>`
"""""""""""""""""""""""""""""""""""

      Algorithm to use for MPn ( :math:`n>2` ) computation (e.g., MP3 or MP2.5 or MP4(SDQ)). See :ref:`Cross-module Redundancies <table:managedmethods>` for details. Since v1.4, default for non-orbital-optimized MP2.5 and MP3 is DF.

      * **Type**: string
      * **Possible Values**: DF, CONV, CD
      * **Default**: CONV
